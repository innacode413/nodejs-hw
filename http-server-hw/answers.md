## 1. Чому при відкритті http://localhost:3000 не потрібен DNS?

localhost — це зарезервоване ім'я для мого власного комп'ютера. Воно має IP-адресу 127.0.0.1 (loopback) і прописане у файлі hosts. Жодних DNS-запитів не відбувається, бо зіставлення localhost ↔ 127.0.0.1 відбувається локально.

## 2. Що станеться, якщо запустити два сервери на одному порту 3000?

Буде помилка EADDRINUSE — "address already in use". Я спробувала, і мені вивело: `Error: listen EADDRINUSE: address already in use :::3000`. Другий сервер не запуститься, бо порт 3000 уже зайнятий першим.

## 3. Для чого потрібен заголовок Content-Type? Що зміниться, якщо його прибрати?

Content-Type підказує браузеру, що саме я йому надсилаю — звичайний текст, HTML, JSON тощо. Якщо його прибрати, браузер може неправильно відобразити сторінку або вгадувати тип самостійно (MIME-сніффінг), що іноді небезпечно.

## 4. У якому місці коду визначається, який код стану поверне сервер?

Код стану задається в методі `res.writeHead()` — перший аргумент це число (200, 404, 500 тощо). У моєму коді я прописую різні коди для різних URL у кожному блоці if/else.
